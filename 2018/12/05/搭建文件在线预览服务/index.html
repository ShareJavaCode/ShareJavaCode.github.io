搭建文件在线预览服务

本文总共1445个字 | 您是第位看到它们的小伙伴

工作环境：mac

####################################################
要求：
https://gitee.com/kekingcn/file-online-preview
研究上面的项目，
1.能够在自己机器或者服务器上搭建成功
2.总结搭建服务文档流程和注意点。

实现：
1.下载项目 git clone https://github.com/kekingcn/file-online-preview.git

2.安装OpenOffice 【我安装在mac，下方网址里还有其他版本的安装包】
http://www.openoffice.org/download/index.html

3.安装redis,redis密码可以去/etc/redis.conf里面设置，然后配置在本项目的application-dev.properties中，也可以不设置

4.在application-dev.properties里面，配置redis地址和OpenOffice目录

redis的端口号如果没有设置，默认是6379
file.dir目录是转换文件实际存储地址，注意要以/结尾
office.home要配置到Contents目录（我的office是安装在mac上的）

5.file.dir配置的目录要跟之前nginx配置的目录一致

6.运行FilePreviewApplication的main方法，服务启动后，访问http://localhost:8012/

7.在?url=后面放入之前用nginx服务器得到的路径即可在线预览，到此配置完成！

[注意]
*我的完整application-dev.properties配置文件

*配置过程中用到的一些命令
启动本地redis服务
sudo /usr/local/bin/redis-server /etc/redis.conf
查看redis进程
ps ax|grep redis
杀死redis进程
sudo kill -9 14605

*运行项目遇到的问题
一开始sigar依赖不进来，是因为maven本地仓库下载的文件不完整，删掉重新下载即可
